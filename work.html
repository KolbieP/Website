Design and Coding Process

When approaching each project, no matter the size, I follow a consistent process that ensures thorough development and exceptional results.

1

Requirements and Planning

For each project, I start by analyzing all necessary variables and components. When working with a client, I take detailed notes on their desired outcomes and specific needs. This phase also involves conducting unique research for each project, allowing me to form a tentative direction for the project's development.

2

Wireframes and System Design

When tackling a complex project, I find it beneficial to simplify the project by creating wireframes. This allows me to visualize the project's bare bones before bringing it to life. By creating wireframes, the coding phase becomes much easier and more time-efficient.

3

Coding and Testing

This step is the most time-consuming and operates like a continuous loop. I create essential components and functions, test and debug them, and repeat the process for each new element. During this phase, I bring wireframes and research to life, transforming them into a fully functioning project.

4

Deployment and Maintenance

This is the final step of my development process. I present the project to the client or a colleague and receive feedback, which I then implement. After making the necessary changes, I deploy the project. As a developer, it is often my responsibility to maintain the project throughout its lifecycle, which I continue to do if required.

Book Search Web Application

I created this book search web application using React and Vite. It allows users to search for books by their titles using the Open Library API. The application returns a list of five books that match the search query and provides the ability to add them to your TBR (To Be Read) list. Once added to the "Books on Deck" section, books can be deleted or marked as read (struck through).
